“Minimal model construction” comes from the idea that a set of statements should be satisfied by the smallest possible structure—no extra entities, no extra distinctions, no added constraints beyond what is explicitly given.

Formally, a model assigns:

a set of entities (nodes),
relations between them,
and identity mappings,

such that all stated predicates are true.

A model is minimal if:

it introduces no additional entities beyond necessity,
it does not split one entity into several unless forced,
it does not add constraints not entailed by the statements.

Application to the puzzle

The sentence contains an explicit identity:

surgeon ≡ father

and a relation:

father → parent-of → boy

plus the utterance:

surgeon says: boy is my son

A minimal model is:

Entities: {boy, father/surgeon}
Relations: parent(father/surgeon, boy)

This model satisfies all predicates. No contradiction appears. No further structure is required.

Schemas as attractors
Frequently encountered structures—like the “surgeon puzzle”—form attractor basins in this space. When partial cues are present (“surgeon,” “boy,” “cannot operate,” “my son”), the model’s state is pulled toward that basin. Completion then proceeds by unfolding the most statistically coherent continuation associated with that region. The puzzle is forgotten by then: the behavioural script (going through the motions, instinctual reply) wins over instead.
